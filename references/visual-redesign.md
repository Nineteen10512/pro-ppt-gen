# Visual Redesign Reference

Use this when a PPT risks becoming flat text blocks. The goal is to turn source material into information-graphic slide structure before rendering.

## Process

1. Decompose content into information blocks.
2. Assign each block a logical role: definition, evidence, comparison, process, risk, action, result.
3. Identify relationships: sequence, cause-effect, contrast, hierarchy, cycle, dashboard/data story.
4. Plan pages: user-requested page count means core narrative pages; cover/summary may be added as structural pages.
5. Layer each page into core, support, and detail information.
6. Design layout from relationship, not from a generic card template.
7. Generate a prompt or semantic JSON plan with structure elements explicitly named.

## Anti-Flat Rule

A flat slide is a page made of equal rectangles where each rectangle contains a title and bullet list. This is disallowed for core narrative pages.

Each core page must include at least two structure elements:

- connectors/arrows
- nodes
- spatial contrast
- ring/cycle
- progress/status
- asymmetry/hero focus
- hierarchy/tree/matrix
- data visualization

## Page Count

If the user asks for N pages, treat N as core narrative pages. Cover, TOC, section, and summary pages are structural pages and may be added unless the user forbids them.

## Confirmation

Before rendering complex decks, show page role, layout structure, structure elements, and anti-flat acceptance line. Then render only after confirmation when the task context requires a review gate.
